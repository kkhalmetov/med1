# QadamAI — спецификация бренда и публичного AI-блока

Статус: `APPROVED`

Дата: 2026-07-21

Основание: пользовательское решение о переименовании после запуска `GET /patients/{id}/short-review`.

## 1. Цель

Переименовать пользовательский продукт из `Qadam` в `QadamAI` и понятно объяснить на русской и казахской главной странице, какую пользу даёт уже подключённый AI-обзор пациента.

Новый блок должен объяснять фактический сценарий: система объединяет сведения об используемых ТСР, ежедневных отчётах, жалобах и истории чата, формирует краткую выжимку и показывает её специалисту. AI не заявляется как средство диагностики и не заменяет решение специалиста.

## 2. Технологический контекст

- Next.js 16 App Router, React 19 и TypeScript.
- `next-intl` для `/ru` и `/kk`.
- Существующая дизайн-система Q: светлый фон, зелёные акценты, скругления и Lucide icons.
- Команды проекта: `pnpm test:e2e`, `pnpm verify`, `pnpm build`.

## 3. Границы изменения

### Входит в scope

- Видимое имя продукта, доступные названия, metadata, manifest и заголовки SVG: `QadamAI`.
- Русский и казахский текст входа, лендинга и ролевых кабинетов, где упоминается бренд.
- Новый содержательный AI-раздел на публичной главной между пользовательским сценарием и футером.
- Обновление регрессионных тестов и основной документации.

### Не входит в scope

- Переименование GitHub-репозитория, Vercel-проекта и production URL.
- Переименование cookie, cache keys, OpenAPI snapshot, имён файлов и внутренних программных идентификаторов.
- Изменение backend-контракта или добавление новых AI-возможностей.
- Обещания диагноза, лечения, прогноза или полностью автоматического решения.

## 4. Пользовательский интерфейс

AI-раздел состоит из двух смысловых зон:

1. Крупный заголовок и короткое объяснение ценности `QadamAI`.
2. Последовательность из трёх шагов: сбор контекста, выделение важного, помощь специалисту.

В разделе явно указано, что окончательное решение остаётся за специалистом. На ширине 320–360 px контент перестраивается в одну колонку без горизонтальной прокрутки. Иконки декоративны; смысл доступен в тексте.

Пример структуры:

```tsx
<section aria-labelledby="ai-overview-title">
  <h2 id="ai-overview-title">ИИ собирает важное в один обзор</h2>
  <ol>
    <li>Собирает контекст</li>
    <li>Выделяет важное</li>
    <li>Помогает специалисту</li>
  </ol>
</section>
```

## 5. Локализация

Русская и казахская версии имеют одинаковую структуру и смысл. Во всех пользовательских строках используется точное написание `QadamAI`. Технические термины API и HTTP на лендинге не показываются.

## 6. Проверка

- RED: Playwright ожидает новый бренд и AI-блок до реализации и падает.
- GREEN: тот же тест проходит после изменения.
- Playwright проверяет RU-копирайт, KK-копирайт, доступные имена и отсутствие overflow.
- `pnpm verify` и `pnpm build` проходят полностью.
- Визуальная проверка desktop и mobile подтверждает читаемость и соответствие существующему стилю.
- Перед выпуском выполняются code review, secret scan и production smoke.

## 7. Критерии готовности

- На пользовательских поверхностях нет устаревшего самостоятельного имени `Qadam`.
- Title, manifest, лого, вход, кабинеты и футер показывают `QadamAI`.
- На `/ru` и `/kk` есть понятный AI-раздел с фактическими источниками выжимки.
- Текст не создаёт медицинских обещаний и указывает на роль специалиста.
- GitHub `main` и Vercel production содержат один проверенный релиз.
