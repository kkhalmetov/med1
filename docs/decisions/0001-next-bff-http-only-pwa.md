# ADR-001: Next.js BFF, HttpOnly-сессия и privacy-safe PWA

## Статус

Принято.

## Дата

2026-07-20.

## Контекст

Qadam должен предоставить один двуязычный web-интерфейс для трёх ролей и полностью реализовать REST-контракт существующего backend. Backend находится на отдельном origin, выдаёт access/refresh tokens и работает с медицинскими данными, фотографиями, CSV и PDF. Решение должно быстро разворачиваться на Vercel, работать на телефонах и устанавливаться как приложение.

Ключевые ограничения:

- browser не должен хранить JWT в доступном JavaScript storage;
- произвольный proxy к backend недопустим;
- персональные данные и API-ответы не должны сохраняться service worker;
- один код должен поддерживать RU/KK, desktop и mobile;
- контракт backend является источником истины для всех 52 операций.

## Решение

Использовать Next.js App Router как frontend и same-origin Backend-for-Frontend:

1. Login/refresh выполняются серверными route handlers; токены хранятся только в `Secure`, `HttpOnly`, `SameSite=Lax` cookies.
2. Универсальный backend route принимает только методы, пути, query-параметры и content types, присутствующие в зафиксированном OpenAPI.
3. Ответ `401` инициирует ровно одну серверную попытку refresh, после чего сессия очищается.
4. OpenAPI snapshot генерирует TypeScript-типы и проверяется на drift в CI.
5. PWA service worker кеширует только versioned framework assets и иконки; HTML, API, media и документы всегда идут через сеть.
6. Ролевой UI использует общий schema-driven workbench, а критичные auth/dashboard сценарии имеют специализированные компоненты.

## Рассмотренные альтернативы

### Чистая SPA с JWT в localStorage

Быстрее в минимальном прототипе, но токены доступны при XSS, сложнее контролировать cross-origin запросы и невозможно скрыть backend origin. Отклонено по соображениям безопасности.

### Прямые browser-запросы к backend с in-memory token

Снижает риск постоянного хранения, но refresh теряется после перезагрузки, остаются CORS/mixed-origin ограничения и токен всё равно доступен JavaScript. Отклонено.

### Нативные iOS/Android приложения

Дают глубокую платформенную интеграцию, но требуют двух delivery pipelines и не соответствуют срокам хакатона. PWA покрывает установку и mobile UX одним кодом.

### Полный offline-first cache

Может улучшить работу без сети, но создаёт локальные копии чувствительных медицинских данных и риск устаревших назначений. Отклонено; offline UX показывает состояние сети без кеширования данных.

## Последствия

- Внешний HTTP backend доступен только серверной инфраструктуре; browser общается с HTTPS origin Vercel.
- Все операции проходят через один контролируемый security boundary.
- Runtime требует Node.js serverless-функций и не может быть чисто статическим export.
- Изменения Swagger требуют обновления snapshot, типов, coverage и UI.
- Service worker остаётся намеренно ограниченным; полноценный offline-режим не является функцией продукта.
