# QadamAI — мобильный чат

Статус: `APPROVED /build-auto`

Дата: 2026-07-21

## Цель

Сделать чат пациента и специалиста удобным на телефонах 320–390 px, не изменяя backend API и семантику отправки текста или фотографии.

## Зафиксированные проблемы

- нижняя навигация перекрывает форму отправки;
- многострочное поле и выбор файла занимают большую часть первого экрана;
- пациенту приходится прокручивать страницу от переписки к отдельной карточке отправки;
- список не открывается на последнем сообщении;
- у специалиста выбранная переписка остаётся ниже полного списка пациентов;
- время сообщения не показывается.

## Требуемое поведение

- Последние сообщения видны сразу; прокручивается только журнал переписки.
- Компактная форма с полем, кнопкой отправки и прикреплением фото остаётся выше нижней навигации.
- Журнал имеет доступную семантику `log` и локализованное имя.
- Сообщение показывает локализованные дату и время; длинный текст и фото не создают overflow.
- После выбора пациента специалист видит только выбранный диалог и кнопку возврата к списку пациентов.
- Desktop-компоновка сохраняет два осмысленных столбца; API-запросы и payload остаются без изменений.

## Проверка

- RED/GREEN Playwright для пациента и специалиста на 360 × 820 px.
- Browser screenshot после исправления на 320/360 px.
- Проверка RU/KK, keyboard/axe, `pnpm verify` и полный role E2E.
- Production smoke после Vercel deployment.
