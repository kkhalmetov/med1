# Changelog

Все заметные изменения Qadam фиксируются в этом файле.

## [0.2.0] - 2026-07-21

### Добавлено

- пациенту доступна только одна отправка ежедневного отчёта за локальный календарный день.

### Изменено

- логотип, favicon и PWA-иконка заменены единым знаком с буквой `Q`;
- с публичной главной удалён блок слогана со щитом.

### Исправлено

- пустой `PATCH` проверки отчёта проходит через BFF без ложной ошибки `415`.

## [0.1.9] - 2026-07-21

### Исправлено

- счётчик чата пациента использует массив непрочитанных сообщений из backend и больше не показывает постоянное тире.

## [0.1.8] - 2026-07-21

### Исправлено

- подтверждено отображение истории статусов после backend-исправления `PatientStatusService.history()`; успешный сценарий закреплён браузерным регрессионным тестом.

## [0.1.7] - 2026-07-20

### Исправлено

- сокращены подсказка ввода ID пациента и сообщение об отсутствии пациента в разделе актов выдачи.

## [0.1.6] - 2026-07-20

### Исправлено

- обращения server-side proxy к backend ограничены десятью секундами вместо зависания до platform timeout;
- сетевые сбои backend получают безопасный correlation ID и структурированную диагностику без токенов, паролей и персональных данных.

## [0.1.5] - 2026-07-20

### Исправлено

- история статусов пациента отличает временную ошибку backend от пустой истории;
- после проверки отчёт сразу обновляется и не оставляет активной повторную кнопку;
- ошибки проверки отчёта и PDF-выгрузки показывают причину по категории ответа backend;
- данные отчёта, кнопки и уведомления получили устойчивые вертикальные отступы;
- поиск актов поясняет, что принимает ID пациента, а не ID специалиста или акта.

## [0.1.4] - 2026-07-20

### Исправлено

- поиск актов выдачи теперь объясняет, что пациент не найден, и предлагает проверить ID из CSV-выгрузки вместо общей ошибки.

## [0.1.3] - 2026-07-20

### Исправлено

- мобильная навигация подсвечивает только текущий раздел кабинета;
- в мобильной шапке добавлена доступная кнопка выхода;
- логотип внутри кабинета ведёт на главную соответствующей роли без выхода из сессии.

## [0.1.2] - 2026-07-20

### Исправлено

- логотип в футере лендинга теперь возвращает к началу страницы;
- ошибки входа показываются одной красной плашкой без дублирования под полями;
- поиск актов выдачи и фильтр наблюдения выровнены в одну строку;
- загрузка фотографий во всех кабинетах приведена к единому стилю Qadam.

## [0.1.1] - 2026-07-20

### Исправлено

- технический Swagger-workbench заменён предметными кабинетами пациента, специалиста и администратора;
- все 52 backend-операции привязаны к контекстным спискам, карточкам, формам, чатам и выгрузкам;
- исправлены лендинг, экран входа, форма пароля и главная пациента по production-ревью;
- диапазон `discomfortLevel` синхронизирован с live OpenAPI: `0–10`;
- добавлены RU/KK product copy, mobile checks и browser mutation tests для трёх ролей.

## [0.1.0] - 2026-07-20

### Добавлено

- двуязычная RU/KK installable PWA с кабинетами `PATIENT`, `PRACTITIONER` и `ADMIN`;
- полный UI и BFF для 52 операций и 30 схем backend OpenAPI;
- ежедневные отчёты, жалобы с фото, чат, изделия, справочники и CSV/PDF;
- HttpOnly-сессии, OpenAPI allowlist и безопасная обработка media/download;
- адаптивная дизайн-система, keyboard navigation и axe-проверки;
- unit/integration, role-based Playwright и live smoke suites;
- CI quality gates, security/code reviews и Vercel runbook.
